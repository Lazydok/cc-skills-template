### 2. 잘못된 할인 로직 및 음수 합계 미검증
*   **심각도:** **High**
*   **위치:** 18라인, 22라인 (`process_order` 함수 내)
*   **설명:** 할인이 전체 금액에서 단순히 차감되는데, 할인이 `price * quantity`보다 클 경우 `item_total` 및 최종 `total`이 음수가 될 수 있습니다. 또한 주석에 언급된 대로 할인이 퍼센트(%)인 경우 현재 로직은 완전히 잘못되었습니다.
*   **수정 제안:** 할인이 퍼센트인지 고정 금액인지 명확히 하고, 합계가 0보다 작아지지 않도록 검증 로직을 추가해야 합니다.

### 3. 부동 소수점 연산 오차 및 하드코딩된 세율
*   **심각도:** **Medium**
*   **위치:** 26라인 (`process_order` 함수 내)
*   **설명:** 통계 및 금융 계산에서 `float` 타입을 직접 사용하면 부동 소수점 오차가 발생할 수 있습니다. 또한 세율(0.1)이 하드코딩되어 있어 유지보수가 어렵습니다.
*   **수정 제안:** 정확한 금액 계산을 위해 `decimal.Decimal` 클래스를 사용하고, 세율은 설정 파일이나 상수로 관리하십시오.
    ```python
    from decimal import Decimal, ROUND_HALF_UP
    # 수정 예시
    tax_rate = Decimal('0.1')
    tax = (total * tax_rate).quantize(Decimal('1'), rounding=ROUND_HALF_UP)
    ```

### 4. 캐시 메모리 누수 위험 (Eviction Policy 부재)
*   **심각도:** **Medium**
*   **위치:** 59라인 (`OrderCache.set` 메서드 내)
*   **설명:** `max_size`가 설정되어 있음에도 불구하고, 캐시가 가득 찼을 때 기존 데이터를 삭제(Eviction)하는 로직이 없습니다. 이는 데이터가 계속 쌓여 메모리 부족(OOM) 오류를 일으킬 수 있습니다.
*   **수정 제안:** 캐시가 가득 차면 가장 오래된 항목을 제거(LRU 등)하는 로직을 추가하십시오.
    ```python
    def set(self, order_id, order):
        if len(self._cache) >= self.max_size:
            # 단순 예시: 첫 번째 항목 삭제
            del self._cache[next(iter(self._cache))]
        self._cache[order_id] = order
    ```
