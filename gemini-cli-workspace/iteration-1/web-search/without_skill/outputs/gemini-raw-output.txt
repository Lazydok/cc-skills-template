Node.js 22(코드네임 'Jod')는 2024년 10월에 정식으로 LTS(Long-Term Support) 단계에 진입했습니다. 공식 문서와 릴리스 노트를 바탕으로 정리한 주요 변경사항과 새로운 기능은 다음과 같습니다.

### 1. 성능 및 엔진 업데이트
*   **V8 엔진 12.4 적용:** 최신 V8 엔진 업데이트를 통해 JavaScript 실행 성능이 향상되었으며, `Array.fromAsync()`, 새로운 `Set` 메서드(intersection, union 등), `Iterator` 헬퍼 등의 최신 문법을 지원합니다.
*   **Maglev 컴파일러 기본 활성화:** V8의 중간 단계 JIT 컴파일러인 Maglev가 기본적으로 활성화되었습니다. 이는 특히 단기간 실행되는 CLI 도구나 초기 실행 단계에서 성능을 크게 개선합니다.
*   **Stream 성능 개선:** `highWaterMark` 기본값이 **16KiB에서 64KiB**로 상향되었습니다. 약간의 메모리 사용량이 늘어날 수 있지만, 데이터 처리가 많은 애플리케이션에서 처리 속도가 전반적으로 향상되었습니다.
*   **AbortSignal 최적화:** `AbortSignal` 인스턴스 생성 속도가 빨라져 `fetch()`나 내장 테스트 러너와 같이 이를 활용하는 API의 오버헤드가 줄어들었습니다.

### 2. 주요 새로운 기능 및 API
*   **기본 WebSocket 클라이언트:** 별도의 외부 라이브러리(예: `ws`) 없이도 브라우저 표준과 호환되는 `WebSocket` 클라이언트를 내장 API로 사용할 수 있게 되었습니다(안정화 단계).
*   **`node --run` 플래그 추가:** `package.json`의 스크립트를 실행하는 새로운 명령입니다. `npm run`보다 훨씬 빠르며, 패키지 매니저를 거치지 않고 직접 스크립트를 실행합니다.
*   **네이티브 Glob 지원:** `node:fs` 모듈에 `glob` 및 `globSync` 함수가 추가되었습니다. 이제 파일 패턴 매칭을 위해 외부 패키지를 사용할 필요가 없습니다.
*   **안정화된 `--watch` 모드:** 파일 변경 시 프로세스를 자동으로 재시작하는 `--watch` 플래그가 이제 안정적인 기능으로 제공됩니다.

### 3. ESM 및 모듈 호환성
*   **`require(esm)` 지원 (실험적):** `--experimental-require-module` 플래그를 통해 CommonJS 환경에서도 동기적 ESM 그래프를 `require()`할 수 있는 기능이 추가되었습니다. 이는 라이브러리 작성자들이 ESM으로 전환하는 데 큰 도움을 줍니다.
*   **ESM 감지 기능 강화:** `package.json`에 `type` 필드가 없더라도 ESM 형식의 파일을 더 똑똑하게 감지합니다.

### 4. 기타 주요 변경사항
*   **내장 테스트 러너 개선:** `node:test` 모듈의 모킹(mocking) 지원이 강화되었고 실행 속도가 더욱 빨라졌습니다.
*   **권한 모델(Permission Model) 고도화:** 파일 시스템, 네트워크, 워커 스레드 등에 대한 접근을 정교하게 제어할 수 있는 실험적 권한 모델이 더욱 정교해졌습니다.
*   **WASI(WebAssembly System Interface) 지원:** WebAssembly 모듈이 시스템 리소스에 안전하게 접근할 수 있는 기능이 개선되었습니다.
